  <action>
**Step 1: Run build.sh to generate production CSS.**

Execute `bash build.sh` from the project root. This downloads the Tailwind CLI binary (if not present) and generates `css/output.css` from `css/input.css --minify`.

If the build fails because the CLI binary download fails, fall back to manually downloading:
```bash
curl -sLO "https://github.com/tailwindlabs/tailwindcss/releases/download/v4.2.2/tailwindcss-macos-arm64"
chmod +x tailwindcss-macos-arm64
./tailwindcss-macos-arm64 -i css/input.css -o css/output.css --minify
```

Verify `css/output.css` exists and is under 15KB.

**Step 2: Create OG image for social sharing.**

Use ImageMagick to create a 1200x630 OG image from one of the converted photos:
```bash
magick assets/images/img_0350.jpg \
  -resize 1200x630^ -gravity center -extent 1200x630 \
  -colorspace sRGB -strip -quality 85 \
  assets/images/og-image.jpg
```

This creates a center-cropped 1200x630 JPEG suitable for LinkedIn/social previews.

**Step 3: Switch index.html from CDN to production CSS.**

In index.html, make these changes:

1. REMOVE the Tailwind CDN `<script>` tag:
   ```html
   <!-- REMOVE THIS LINE: -->
   <script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>
   ```

2. REMOVE the `<style type="text/tailwindcss">` block (the entire block including the @import, @theme, and @layer definitions).

3. ADD a CSS link BEFORE the Google Fonts link:
   ```html
   <link href="css/output.css" rel="stylesheet">
   ```

4. UPDATE the OG meta tags to their final values:
   ```html
   <meta property="og:title" content="G&amp;P - Dramatische Leistungssteigerung fuer produzierende Unternehmen">
   <meta property="og:description" content="Beratung fuer produzierende Unternehmen. ERP &amp; KI, Lean, Organisation. Sch(B)rechstunde KI: Freitags 16:00 Uhr.">
   <meta property="og:type" content="website">
   <meta property="og:image" content="assets/images/og-image.jpg">
   <meta property="og:locale" content="de_DE">
   ```

5. VERIFY all asset paths are relative (start with `assets/`, `css/`, `js/` -- NEVER with `/`). Scan the entire file for paths starting with `/` that reference local assets and fix them.

6. Verify `.nojekyll` file exists in project root (created in plan 01).

Per TECH-01: relative paths for GitHub Pages. Per TECH-02: complete OG tags. Per TECH-03/TECH-04: production CSS eliminates 200KB+ CDN overhead.
