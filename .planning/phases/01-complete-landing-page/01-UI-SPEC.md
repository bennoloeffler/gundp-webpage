## Typography

All sizes use Tailwind's built-in scale. The system is mobile-first; desktop overrides use `md:` and `lg:` prefixes.

| Role | Mobile Size | Desktop Size | Weight | Line Height | Font | Tailwind Classes |
|------|-------------|--------------|--------|-------------|------|------------------|
| Display (hero H1) | 36px (`text-4xl`) | 72px (`lg:text-7xl`) | 700 (`font-bold`) | 1.1 (`leading-tight`) | Space Grotesk | `font-heading text-4xl lg:text-7xl font-bold leading-tight` |
| Heading (section H2) | 30px (`text-3xl`) | 48px (`lg:text-5xl`) | 700 (`font-bold`) | 1.2 (`leading-tight`) | Space Grotesk | `font-heading text-3xl lg:text-5xl font-bold leading-tight` |
| Subheading (H3) | 24px (`text-2xl`) | 30px (`lg:text-3xl`) | 500 (`font-medium`) | 1.3 | Space Grotesk | `font-heading text-2xl lg:text-3xl font-medium` |
| Body | 18px (`text-lg`) | 20px (`md:text-xl`) | 400 (`font-normal`) | 1.6 (`leading-relaxed`) | Inter | `font-body text-lg md:text-xl font-normal leading-relaxed` |
| Body emphasis | 18px (`text-lg`) | 20px (`md:text-xl`) | 600 (`font-semibold`) | 1.6 (`leading-relaxed`) | Inter | `font-body text-lg md:text-xl font-semibold leading-relaxed` |
| CTA button | 18px (`text-lg`) | 20px (`md:text-xl`) | 700 (`font-bold`) | 1.0 | Space Grotesk | `font-heading text-lg md:text-xl font-bold` |
| Nav link | 16px (`text-base`) | 16px (`text-base`) | 500 (`font-medium`) | 1.0 | Inter | `font-body text-base font-medium` |

Additional typography rules:
- `lang="de"` on `<html>` element -- required for correct hyphenation
- `hyphens: auto` on body text containers -- prevents German compound word overflow
- `text-balance` on headings -- prevents orphaned words on line breaks
- Letter spacing on Display: `-0.02em` (`tracking-tight`) for tighter hero headlines
- Maximum body text width: `max-w-prose` (65ch) for readability

  --color-gp-gold-hover: #d4b76a;
  --font-heading: 'Space Grotesk', sans-serif;
  --font-body: 'Inter', sans-serif;

```

### Color Distribution (60 / 30 / 10)

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `gp-black` | #0a0a0a | Page background, hero background, main surface |
| Secondary (30%) | `gp-dark` / `gp-charcoal` | #141414 / #1a1a1a | Alternating section backgrounds, card surfaces, nav background (on scroll), footer |
| Accent (10%) | `gp-red` | #c41e3a | See reserved list below |
| Text primary | `gp-white` | #fafafa | Headings, body text, nav links |
| Text secondary | `gp-light` | #e5e5e5 | Subheadings, descriptions, secondary copy |
| Text muted | `gp-grey` | #666666 | Legal text, footer secondary links, captions |
| Highlight | `gp-gold` | #c5a55a | See reserved list below |

### Accent Reserved For (gp-red #c41e3a)

- Primary CTA buttons (background)
- Primary CTA button hover state (gp-red-hover #d6263f)
- Hero headline emphasis word (one word highlighted in red)
- Active nav link indicator (underline or dot)
- Mobile hamburger menu icon

Accent is NEVER used for: body text, section backgrounds, borders, card surfaces, or decorative elements.

### Gold Reserved For (gp-gold #c5a55a)

- Sch(B)rechstunde section accent (section border or heading highlight)
- Pillar card icon/number highlights
- Credibility section metric numbers
- Subtle divider lines between sections

Gold is NEVER used for: buttons, body text, backgrounds, or nav elements.

```

Hero:           gp-black + darkened background image overlay

### Contrast Ratios (WCAG AA Compliance)

| Foreground | Background | Ratio | Pass |
|------------|------------|-------|------|
| #fafafa on #0a0a0a | white on black | 19.4:1 | AAA |
| #fafafa on #141414 | white on dark | 17.9:1 | AAA |
| #fafafa on #c41e3a | white on red (CTA) | 5.2:1 | AA |
| #c41e3a on #0a0a0a | red on black | 3.7:1 | AA Large Text only |

Note: Red text (#c41e3a) on dark backgrounds only passes for large text (18px+ bold or 24px+ regular). Use red for headings and CTA buttons only, never for small body text.

### Empty States

This is a static landing page -- there are no dynamic data-dependent empty states. All content is hardcoded.

### Error States

| Scenario | Handling |
|----------|----------|
| Email client not configured (mailto click) | The `mailto:` link is the only interactive element. If the user has no email client, the browser handles the error natively. No custom error state needed. |
| Images fail to load | All `<img>` tags have descriptive German `alt` text. Layout does not collapse (images have explicit `width`/`height` attributes). |
| JavaScript disabled | Page is fully readable without JS. Scroll animations simply do not play. Nav works via anchor links. No "enable JavaScript" banner needed. |
| Fonts fail to load | System sans-serif fallback renders immediately (declared in font stack). Page remains readable. |

### Destructive Actions

None. This is a read-only landing page with no user-modifiable state.

## Animation Contract

GSAP 3.14.2 + ScrollTrigger for scroll-driven reveals. CSS transitions for hover states.

### Scroll Reveal Animations

| Section | Animation | Duration | Easing | Trigger |
|---------|-----------|----------|--------|---------|
| Hero headline | Fade up from y: 40px, staggered words | 0.8s per word, 0.15s stagger | `power3.out` | Page load (immediate, no scroll trigger) |
| Hero subheadline | Fade up from y: 30px | 0.6s | `power2.out` | Page load, 0.4s delay after headline |
| Hero CTA | Fade up from y: 20px | 0.5s | `power2.out` | Page load, 0.8s delay after headline |
| Problem statements | Fade up from y: 30px, staggered | 0.6s each, 0.2s stagger | `power2.out` | ScrollTrigger, top of element hits 80% viewport |
| Pillar cards | Fade up from y: 40px, staggered left-to-right | 0.7s each, 0.15s stagger | `power3.out` | ScrollTrigger, top of container hits 75% viewport |
| Credibility metrics | Count-up from 0 to target number | 1.5s | `power1.inOut` | ScrollTrigger, element enters viewport |
| Credibility photo | Fade in from opacity 0, slight scale from 0.95 | 0.8s | `power2.out` | ScrollTrigger, 80% viewport |
| Schbrechstunde section | Fade up from y: 30px | 0.6s | `power2.out` | ScrollTrigger, 80% viewport |
| Final CTA | Fade up from y: 30px | 0.6s | `power2.out` | ScrollTrigger, 80% viewport |

### Hover/State Transitions (CSS only, no GSAP)

| Element | Property | Duration | Easing |
|---------|----------|----------|--------|
| CTA buttons | background-color, transform (scale 1.02) | 200ms | ease-out |
| Nav links | opacity (0.7 to 1.0) | 150ms | ease |
| Pillar cards | transform (translateY -4px), box-shadow | 300ms | ease-out |
| Footer links | color (gp-grey to gp-white) | 150ms | ease |

### Sticky Nav Transition

| State | Background | Transition |
|-------|------------|------------|
| Top of page (scrollY <= 80) | transparent | 300ms ease |
| Scrolled (scrollY > 80) | gp-dark with 95% opacity (`bg-gp-dark/95 backdrop-blur-sm`) | 300ms ease |

### Animation Principles

- **Respect `prefers-reduced-motion`**: If user has reduced motion enabled, disable all GSAP animations and set all CSS transitions to 0ms. Elements render in their final state immediately.
- **Animate once only**: Each scroll-triggered animation plays once. After an element is revealed, it stays revealed. No reverse animations on scroll-up.
- **No parallax**: Static backgrounds only. Parallax causes jank on mobile Safari and feels dated.
- **Progressive enhancement**: Page is fully functional and readable without any animations.

## Interaction Contract

### mailto CTA Behavior

All CTAs use `mailto:` links. No JavaScript forms, no backend.

| CTA Location | mailto | Subject Line |
|--------------|--------|--------------|
| Hero | `mailto:f.gloebl@g-und-p.de` | (none -- keep simple) |
| Sch(B)rechstunde | `mailto:f.gloebl@g-und-p.de?subject=Anmeldung%20Sch(B)rechstunde%20KI` | Pre-filled subject |
| Final CTA | `mailto:f.gloebl@g-und-p.de` | (none) |
| Repeated rhythm CTAs | `mailto:f.gloebl@g-und-p.de` | (none) |

### Smooth Scroll

- All `<a href="#section-id">` links use `scroll-behavior: smooth` (CSS)
- `scroll-padding-top: 5rem` (80px) to offset sticky nav
- JavaScript enhancement: `scrollIntoView({ behavior: 'smooth', block: 'start' })` for programmatic scrolling

### Mobile Menu

- Hamburger icon toggles full-screen overlay
- Overlay background: `gp-black` at 98% opacity with `backdrop-blur-sm`
- Menu items centered vertically, large tap targets (48px min height)
- Close on link click (smooth scroll to section, overlay closes)
- Close on outside click or dedicated close button (X icon, top-right)
- Body scroll locked when menu is open (`overflow: hidden` on `<body>`)

### Cookie Banner

- Fixed to bottom of viewport (`fixed bottom-0 inset-x-0`)
- Background: `gp-charcoal` with slight border-top in `gp-slate`
- Single "Verstanden" dismiss button
- Dismissal stored in `localStorage` key `gp-cookie-notice-dismissed`
- If dismissed, banner does not show on subsequent visits
- No opt-in/opt-out needed (Google Fonts is a functional external resource, not tracking)
